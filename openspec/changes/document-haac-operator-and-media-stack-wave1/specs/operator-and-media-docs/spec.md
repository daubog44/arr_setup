## ADDED Requirements

### Requirement: Detailed operator and media documentation is repo-managed

The repo MUST provide durable documentation for bootstrap, media automation, and security behavior beyond a short quick-start.

#### Scenario: The operator needs to understand the request-to-playback flow

- **WHEN** the operator reads the repo documentation
- **THEN** it MUST explain how Seerr, Prowlarr, Radarr, Sonarr, Lidarr, qBittorrent, NAS storage, and Jellyfin interact in the supported stack
- **AND** it MUST explain the supported rerun and recovery paths
